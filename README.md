**داستان این نسخه چیه؟** 
هدف اصلی نسخه ECO اینه که همون امنیت سفت‌وسخت v1.3 رو داشته باشیم، ولی به سرورهای ورسل فشار نیاریم. تو این نسخه عمداً سرعت رو کنترل (Throttle) کردیم تا مصرف رم و پردازنده پایین بمونه و هزینه‌ها یا ریسک لیمیت شدن اکانت کم بشه.

## ✨ تو نسخه ECO چه خبره؟

- ⏱️ **کنترل تایم‌اوت:** `UPSTREAM_TIMEOUT_MS` رو پیش‌فرض گذاشتیم روی `120000` که واسه کانکشن‌های طولانی جواب بده.
- 🛡️ **بستن متدهای اضافه:** فقط با `GET`، `HEAD` و `POST` کار می‌کنه تا امنیت بالاتر بره.
- 🧹 **تمیزکاری هدرها:** هدرهای اضافی پلتفرم و hop-by-hop رو بی‌رحمانه فیلتر می‌کنیم.
- 🔑 **قفل امنیتی:** احراز هویت فقط از طریق هدر `x-relay-key` انجام میشه (توی کوئری دیگه قبول نمی‌کنه).
- 🛣️ **مسیر سفت‌وسخت:** متغیر `RELAY_PATH` اجباریه. اگر درست ست نشه کلا کانکشن بسته میشه.
- 🐛 **لاگ‌های به درد بخور:** لاگ‌های مربوط به تایم‌اوت، ارورها و مدت‌زمان کانکشن رو واضح‌تر کردیم تا دیباگ راحت بشه.
- 🐢 **سرعت دست‌ساز:** روی آپلود و دانلود محدودیت سرعت (Throttling) واقعی گذاشتیم.
- 📉 **دوست‌دار منابع:** کلا بردیمش روی Node runtime با لیمیت ۱۲۸ مگابایت رم و مدیریت کانکشن‌های همزمان.

---

## 🧠 سیستم چطور کار می‌کنه؟

این برنامه روی سرورلس‌های Node اجرا میشه و سه تا لایه اصلی داره:

۱. **لایه امنیت:** همون هسته v1.3 هست که مسیر، متدها و پسورد رو چک می‌کنه.
۲. **لایه کنترل منابع:** با `MAX_INFLIGHT` نمی‌ذاریم یهو صدتا درخواست همزمان بریزه رو سر سرور و کرش کنه.
۳. **لایه کنترل ترافیک:** با `MAX_UP_BPS` و `MAX_DOWN_BPS` پهنای باند رو محدود می‌کنیم. نتیجه‌اش میشه یه سرعت پایدار اما با مصرف خیلی کمتر.

## ⚙️ راهنمای دیپلوی دستی (بدون گیت‌هاب)

### ۱. نصب ابزار ورسل
اول باید ابزار خط فرمان ورسل رو نصب کنی:
```bash
npm i -g vercel
```

### ۲. ورود به اکانت
```bash
vercel login
```

### ۳. شروع دیپلوی
تو پوشه‌ای که فایل‌ها رو اکسترکت کردی این دستور رو بزن و با زدن Enter مراحل رو رد کن:
```bash
vercel deploy
```

### ۴. تنظیم متغیرها (ENV)
حالا برو تو سایت ورسل، تو بخش `Settings -> Environment Variables` پروژه‌ات، این مقادیر رو ست کن:

| متغیر | وضعیت | چیکار میکنه؟ | مقدار پیش‌فرض |
| :--- | :---: | :--- | :--- |
| `TARGET_DOMAIN` | 🔴 اجباری | آدرس سرور اصلیت (Upstream) | - |
| `RELAY_PATH` | 🔴 اجباری | مسیر اختصاصی خودت (مثلا `/api`) | - |
| `RELAY_KEY` | ⚪ اختیاری | همون پسوردی که تو کلاینت ست می‌کنی | (خالی) |
| `UPSTREAM_TIMEOUT_MS` | ⚪ اختیاری | زمان قطعی کانکشن | `120000` |
| `MAX_INFLIGHT` | ⚪ اختیاری | تحملِ تعداد درخواست همزمان | `24` |
| `MAX_UP_BPS` | ⚪ اختیاری | سقف سرعت آپلود (بایت بر ثانیه) | `1572864` |
| `MAX_DOWN_BPS` | ⚪ اختیاری | سقف سرعت دانلود (بایت بر ثانیه) | `1572864` |

### ۵. دیپلوی نهایی
بعد از ذخیره متغیرها، این دستور رو بزن تا تغییرات اعمال بشه:
```bash
vercel --prod
```

## 🧩 تنظیمات به زبون ساده

اگه گیج شدی که اعداد چطور کار می‌کنن، این مثال‌ها رو ببین:

**تنظیمات ۴۸ کانکشن + سرعت ۱.۵ مگابایت:**
- مثل یه خیابون با لاین کمتر و سرعت مجاز پایین‌تره.
- منابع خیلی کمی مصرف می‌کنه.
- سرعت کاربرها کمه.
- اگه یهو شلوغ بشه، احتمال اینکه ارور ۵۰۳ بدن بالاست.

**تنظیمات ۱۲۸ کانکشن + سرعت ۳ مگابایت:**
- مثل یه اتوبان با لاین‌های بیشتر و سرعت بالاتره.
- ترافیک خیلی راحت رد میشه و ارور ۵۰۳ کمتر می‌بینی.
- سرعت بالاست.
- اما مصرف منابعت هم میره بالا.

**فرمول کلی:**
- متغیر `MAX_INFLIGHT` = تعداد لاین‌های خیابون
- متغیرهای `MAX_UP_BPS` و `MAX_DOWN_BPS` = سرعت مجاز تو هر لاین

## 🎚️ تنظیمات آماده (Presets)

این اعداد صرفاً پیشنهادن و بسته به سرور و اینترنت ممکنه کمی فرق کنن. همونی که به کارت میاد رو کپی کن.

### ۱. تمرکز روی سرعت (برای تک‌کاربر یا مصرف کم)
```text
MAX_INFLIGHT=16
MAX_UP_BPS=1310720
MAX_DOWN_BPS=1310720
```

### ۲. حالت متعادل (بهترین بالانس سرعت و مصرف)
```text
MAX_INFLIGHT=48
MAX_UP_BPS=1572864
MAX_DOWN_BPS=1572864
```

### ۳. حالت ECO (تعداد یوزر بالا، سرعت پایین - هدف اصلی این پروژه)
```text
MAX_INFLIGHT=128
MAX_UP_BPS=786432
MAX_DOWN_BPS=786432
```

### ۴. شلوغی زیاد (سرعت فدای پایداری کانکشن)
```text
MAX_INFLIGHT=192
MAX_UP_BPS=655360
MAX_DOWN_BPS=655360
```

### ۵. بقا! (کم‌مصرف‌ترین حالت، فقط واسه اینکه وصل باشن)
```text
MAX_INFLIGHT=256
MAX_UP_BPS=524288
MAX_DOWN_BPS=524288
```

## 🧭 کدومش رو انتخاب کنم؟

- **سرعت تک یوزر برام مهمه:** برو سراغ حالت ۱ یا ۲.
- **یوزر زیاد دارم سرعت مهم نیست:** برو سراغ حالت ۳ یا ۴.
- **فقط می‌خوام تلگرامشون باز بشه:** حالت ۵ جوابه.

---

## 🧪 چطور خودم تنظیمش کنم؟ (تیونینگ)

۱. **ارور ۵۰۳ زیاد می‌گیری؟** یعنی ترافیک گیر کرده. باید `MAX_INFLIGHT` رو پله‌پله ببری بالا (مثلا از ۴۸ بکنی ۶۴ بعد ۹۶ بعد ۱۲۸).
۲. **سرعت کمه؟** مقدار `MAX_DOWN_BPS` رو ۱۰ تا ۲۰ درصد زیاد کن.
۳. **ورسل داره گیر میده منابع زیاد مصرف کردی؟** مقدار `MAX_DOWN_BPS` رو ۱۰ تا ۲۰ درصد کم کن.

## 💻 نمونه کانفیگ برای کلاینت (v2ray / xray)
```text
vless://UUID-HERE@vercel.com:443?encryption=none&security=tls&sni=vercel.com&fp=chrome&alpn=h2&insecure=0&allowInsecure=0&type=xhttp&host=YOUR-VERCEL-DOMAIN&path=%2Fapi&mode=auto#XHTTP-ECO
```
*(یادت نره دامنه ورسل خودت رو جایگزین کنی و `path` رو همون چیزی بذاری که تو `RELAY_PATH` ست کردی. `%2F` همون کاراکتر اسلش `/` هست).*

## 🛠️ معنی ارورها (Status Codes)

- `200` : همه‌چی عالیه و وصلی.
- `403` : پسوردی که تو `x-relay-key` زدی غلطه.
- `404` : مسیری که تو کلاینت زدی با `RELAY_PATH` هم‌خونی نداره.
- `405` : متد درخواستی غیرمجازه.
- `500` : متغیرهای محیطی (ENV) رو درست ست نکردی.
- `502` : تونل نمی‌تونه به سرور اصلیت (Upstream) وصل بشه.
- `503` : ترافیک همزمان بالاست و به سقف `MAX_INFLIGHT` رسیدی.
- `504` : سرور اصلیت جواب نمیده (تایم‌اوت).

---

## 🔐 اگه RELAY_KEY ست کردیم چیکار کنیم؟

اگر تو تنظیمات ورسل بهش کلید دادی، کلاینتت حتما باید هدر زیر رو بفرسته، وگرنه وصل نمیشه:
```text
x-relay-key: YOUR_SECRET_KEY
```
